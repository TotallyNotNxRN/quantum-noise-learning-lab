# Quantum Noise Learning Lab

> An interactive educational lab for exploring how a single qubit decoheres
> under noise — and what we can do about it.

A focused, single-qubit teaching tool: density matrices, three Kraus
channels, fidelity, purity, eigenvalue analysis, side-by-side analytical
validation, and a classical-style repetition-code intuition for why
quantum error correction exists. Built as a Streamlit interface on top of
a NumPy/SciPy engine that is covered by **211 automated tests**.

## Why this project exists

Most introductions to quantum decoherence are walls of LaTeX. This lab
takes the opposite approach: every equation is wired to a slider, and
every visualization is generated by the same engine the tests cover.
The Validation Lab compares the simulator to hand-derived closed-form
formulas for amplitude and phase damping of `|+⟩` and keeps the
agreement below `1e-12` across the full parameter range — so the math
never has to be taken on faith.

## Who it is for

- Undergraduates learning quantum information for the first time.
- Self-learners who want intuition for what density matrices and Kraus
  channels actually *do* to a state.
- Course instructors looking for a small, focused supplement to a textbook
  introduction to noise and decoherence.

It is intentionally **not** a general-purpose quantum-computing simulator.
See [Limitations](docs/limitations.md).

## Key features

- **Five focused modules.** Foundations, Noise, Metrics, Validation,
  Protection. Each has its own learning objective, sidebar controls, and
  visualizations, and connects directly to the next one.
- **Three noise channels.** Amplitude damping, phase damping (random-Z
  convention), depolarizing. Kraus operators are rendered in LaTeX and
  annotated with the physical event they model.
- **Live metrics.** Fidelity, purity, eigenvalues, trace, Hermiticity, and
  PSD validity update as you move the sliders. Tooltips show the numerical
  tolerance for every check.
- **Analytical validation.** Closed-form `ρ_AD(γ)`, `F_AD(γ)`, `ρ_PD(λ)`,
  `F_PD(λ)` are overlaid on the simulator. Tests enforce
  `|F_sim − F_ana| ≤ 1e-12` across the full grid; the Validation Lab
  shows the agreement live.
- **Honest QEC intuition.** A classical-style 3-repetition code for
  bit-flip errors only, with an explicit panel explaining what it does
  **not** address (phase errors, syndromes, no-cloning).
- **Reproducible.** Editable install, deterministic engine, no cloud
  calls, no auth, no database, no hidden state.

## Scientific background

This lab is built on three foundational ideas from open quantum systems.
A pedagogical introduction lives in
[`docs/01_foundations.md`](docs/01_foundations.md); the items below link to
deeper, focused notes:

1. **Density matrices** generalize state vectors to handle statistical
   mixtures. Any 2×2 Hermitian, positive-semidefinite, trace-1 matrix is a
   valid single-qubit state. See [`docs/02_density_matrices.md`](docs/02_density_matrices.md).
2. **Kraus operators** are the standard representation of completely
   positive trace-preserving (CPTP) maps:
   `ℰ(ρ) = Σᵢ Eᵢ ρ Eᵢ†` with `Σᵢ Eᵢ† Eᵢ = I`. See
   [`docs/03_noise_channels.md`](docs/03_noise_channels.md).
3. **Squared Uhlmann fidelity** quantifies how close two quantum states
   are: `F(ρ, σ) = (Tr √(√ρ σ √ρ))²`. Purity and eigenvalues complement it.
   See [`docs/04_metrics.md`](docs/04_metrics.md).

## Main equations

Parameters: `γ` (amplitude-damping probability of decay), `λ` (phase-damping
strength in the random-Z convention), `p` (depolarizing probability), all in
`[0, 1]`. The Validation Lab's analytical guarantee is specifically for the
initial state `|+⟩`; other initial states are still simulated faithfully but
without a closed-form check.

**Convention note:** phase-damping parameterizations differ across textbooks.
This project uses the random-Z form, which gives off-diagonals scaled by
exactly `(1 − λ)`. See [`docs/03_noise_channels.md`](docs/03_noise_channels.md)
for context.

**Amplitude damping** of `|+⟩` (closed form):

$$
\rho_{AD}(\gamma) = \begin{pmatrix}
(1+\gamma)/2 & \sqrt{1-\gamma}/2 \\
\sqrt{1-\gamma}/2 & (1-\gamma)/2
\end{pmatrix},
\qquad
F_{AD}(\gamma) = \frac{1 + \sqrt{1-\gamma}}{2}.
$$

**Phase damping** of `|+⟩` (random-Z convention):

$$
\rho_{PD}(\lambda) = \begin{pmatrix}
1/2 & (1-\lambda)/2 \\
(1-\lambda)/2 & 1/2
\end{pmatrix},
\qquad
F_{PD}(\lambda) = 1 - \lambda/2.
$$

**Depolarizing**: `D(ρ) = (1−p) ρ + p I/2`.

**Repetition-code success** (classical-style, bit-flip only):

$$
P_{unprot}(p) = 1 - p,\qquad
P_{prot}(p) = (1-p)^3 + 3p(1-p)^2.
$$

Derivations and convention notes live in [`docs/`](docs/).

## Installation

Python 3.10+ recommended. From a clean clone:

```bash
git clone https://github.com/USER/quantum-noise-learning-lab.git
cd quantum-noise-learning-lab

# (optional but recommended) create a virtual environment
python -m venv .venv
source .venv/bin/activate                # macOS/Linux
# .venv\Scripts\activate                 # Windows PowerShell

# install the engine + all UI/test dependencies needed by the commands below
pip install -e ".[all]"
```

`pip install -e ".[all]"` installs the engine **and** Streamlit, Plotly,
Matplotlib, and pytest. The minimal `pip install -e .` installs only the
engine (NumPy + SciPy); the commands further down (`pytest`, `streamlit
run`, the example scripts) require the `[all]` extra.

Replace `USER` in the clone URL above with the actual GitHub owner of your
fork.

## Running the app

```bash
streamlit run app/main.py
```

Streamlit prints a local URL (default `http://localhost:8501`). Use the
sidebar to navigate between the five labs.

## Running the tests

```bash
pytest -q
```

The full suite has 211 tests covering: state construction, density-matrix
validity, all three channels at intermediate parameter values, Kraus
completeness, fidelity / purity / eigenvalue bounds, analytical-vs-simulated
agreement to `1e-12`, repetition-code formulas, the engine/UI architecture
boundary, and a clean-clone smoke test.

## Example scripts

```bash
python examples/run_single_qubit_sweep.py   # CSV table to stdout + results/data/
python examples/generate_all_figures.py     # PNG figures to results/figures/
```

Both run headless against the engine and are useful for reproducing
portfolio figures without needing the UI.

## Limitations

See [`docs/limitations.md`](docs/limitations.md). Briefly: single-qubit
only, three noise channels, classical-style repetition code (not real QEC),
no hardware execution, no fault tolerance. These constraints are
intentional — the project optimizes for depth, not feature count.
